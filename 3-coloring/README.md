# Отчёт по задаче Coloring

## Как запустить:

Чтобы запустить основное решение, выполните команду `./run-all.sh` из директории `./3-coloring`.

## Чекер:

Запускается скриптом `./run-all.sh`. Код: `./src/checker.py`. Отчёты расположены в файлах `verdict.txt` в директориях `./results*`.

## Где искать отчёт:

- `./results-simple`: ответы и отчёт алгоритма без отсекающих эвристик

# Описание алгоритмов

Я решил начать решение задачи с простого алгоритма без сложных эвристик. Моя программа перебирает все вершины графа, смотрит для очередной вершины, какие её соседи уже покрашены, и красит вершину в первый неиспользованный цвет (MEX цветов соседей). Вершины я предварительно посортировал по убыванию степени. Это решение проходит первые пороги на всех тестах, кроме первых двух.

Чтобы улучшить решение, я запускаю его много раз, каждый раз перешаффливая вершины (`greedy_randomized`). Это решение пробивает первые пороги уже на всех тестах.

По совету Виталия, я попробовал перебирать порядок покраски вершин генетикой (`genetics.hpp`). Запуск этого алгоритма с таймаутом на минуту (`./util/Stopwatch.hpp`) позволил мне достичь второго порога на тесте 50, но в целом мой генетический алгоритм оказался слабым, сопоставимым по качеству ответов с рандомом с перешаффливанием.

Затем я решил реализовать перебор с отсечениями (`./branching.hpp`). Чтобы перебор чаще отсекался, я решил перебирать вершины начиная с тех, у которых в текущей раскраске много разноцветных соседей (как в алгоритме `DSatur` из книги, которую Михаил кидал в чат). Чтобы не считать число разноцветных соседей каждый раз за линию, я реализовал структуру данных, поддерживающую их динамически (`DSaturAwareGraph`). Перед тем, как писать непосредственно бранчинг, я решил протестировать структуру алгоритмом DSatur (`d_satur`). Этот алгоритм достиг второго порога на тесте 500 и существенно улучшил ответ на тесте 1000.

Бранчинг с таймаутом на минуту достигает второго порога на всех тестах, кроме 250 и 1000.
